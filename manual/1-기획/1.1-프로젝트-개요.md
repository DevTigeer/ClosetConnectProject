# 프로젝트 개요 (Project Overview)

**작성일**: 2024-12-30
**작성자**: Development Team
**버전**: 1.0

---

## 📌 요약

ClosetConnect는 **패션 클로젯 관리 및 소셜 플랫폼**으로, 사용자가 자신의 옷을 디지털화하여 관리하고, 커뮤니티를 통해 패션 정보를 공유하고, 중고 의류 거래까지 할 수 있는 통합 패션 라이프 플랫폼입니다.

---

## 1. 프로젝트 목적

### 비즈니스 목적
- **문제**: 사람들은 자신이 가진 옷을 정확히 파악하지 못해 비슷한 옷을 중복 구매하거나, 코디에 어려움을 겪음
- **해결**: AI 기반 옷장 디지털화 및 자동 코디 추천으로 옷 활용도 증가 및 구매 최적화
- **차별점**: 단순 옷장 관리가 아닌 **AI 처리 + 커뮤니티 + 중고거래**를 통합한 패션 생태계

### 기술 목적
- AI 이미지 처리 기술 (배경 제거, 세그멘테이션, 인페인팅) 실전 적용
- 비동기 메시지 큐 시스템(RabbitMQ)을 활용한 AI 처리 파이프라인 구축
- Spring Boot + React + Python AI 서버 멀티 스택 아키텍처 설계

---

## 2. 타겟 사용자

### 주요 타겟
1. **20-30대 패션 관심층**
   - 옷이 많지만 코디가 어려운 사람
   - SNS에 OOTD(Outfit Of The Day)를 공유하는 사람

2. **미니멀리스트**
   - 옷장을 정리하고 싶은 사람
   - 필요 없는 옷을 중고로 판매하고 싶은 사람

3. **패션 초보자**
   - 날씨에 맞는 옷 추천이 필요한 사람
   - AI 코디 추천을 받고 싶은 사람

---

## 3. 해결하려는 문제

### 문제 정의
1. **옷장 파악 어려움**: 옷이 많은데 어떤 옷이 있는지 기억 안 남
2. **코디 고민**: 매일 아침 옷 고르는데 시간이 오래 걸림
3. **중복 구매**: 비슷한 옷을 또 사는 경우 발생
4. **안 입는 옷 처리**: 옷장에 방치된 옷 처리 방법 부족
5. **날씨 대응**: 날씨에 맞는 옷을 선택하기 어려움

### 솔루션
1. **AI 기반 옷장 디지털화**: 사진만 찍으면 자동으로 배경 제거 및 분류
2. **AI 코디 추천**: 여러 옷을 조합한 가상 입기 이미지 생성
3. **OOTD 저장소**: 마음에 드는 코디를 저장하고 관리
4. **날씨 기반 추천**: 현재 날씨에 맞는 옷 추천
5. **중고거래 연동**: 안 입는 옷을 바로 판매
6. **커뮤니티**: 패션 정보 공유 및 소통

## 4. 성공 기준 (KPI)

### 정량 지표
| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| MAU (월 활성 사용자) | 1,000명 | 로그인 수 |
| 옷 등록 수 | 5,000개 | DB 레코드 수 |
| AI 코디 생성 수 | 500회/월 | API 호출 수 |
| OOTD 저장 수 | 200개/월 | DB 레코드 수 |
| 중고거래 성사율 | 30% | 주문 완료 / 상품 등록 |
| 평균 세션 시간 | 5분 이상 | 분석 도구 |

### 정성 지표
- 사용자 만족도 설문 4점 이상 (5점 만점)
- AI 코디 결과물 품질 만족도 80% 이상

## 5. 프로젝트 범위

### ✅ In Scope (포함)
- **옷장 관리**: 옷 등록, 조회, 삭제, 카테고리 분류
- **AI 처리**: 배경 제거(rembg), 세그멘테이션, 인페인팅
- **AI 코디**: Gemini AI를 활용한 가상 착용 이미지 생성
- **OOTD**: 코디 결과 저장 및 갤러리
- **날씨 추천**: 현재 날씨 기반 옷 추천
- **커뮤니티**: 게시판, 댓글, 좋아요
- **중고거래**: 상품 등록, 채팅, 결제(TossPayments), 주문 관리

### ❌ Out of Scope (제외)
- 실제 쇼핑몰 연동 (Phase 2)
- AR 가상 피팅 (Phase 2)
- AI 스타일 분석 및 추천 (Phase 2)
- 모바일 앱 개발 (Phase 2, 웹 우선)

## 6. 주요 기술 스택

### Backend
- Spring Boot 3.5.7 (Java 17)
- Spring Security + JWT
- Spring Data JPA / Hibernate
- MariaDB
- RabbitMQ (메시지 큐)

### Frontend
- React + Vite
- React Router
- Axios
- WebSocket (STOMP)

### AI Server
- Python 3.x
- FastAPI
- Google Gemini AI (이미지 생성)
- rembg (배경 제거)
- U2Net (세그멘테이션)
- LaMa (인페인팅)

### Infrastructure
- Gradle (빌드)
- Git (버전 관리)

## 8. 리스크 및 제약사항

### 기술 리스크
1. **AI 처리 속도**: 이미지 처리 시간이 오래 걸릴 수 있음
   - **대응**: 비동기 처리(RabbitMQ) + 진행도 표시

2. **Gemini AI 비용**: API 호출 비용 증가 가능성
   - **대응**: 호출 수 제한, 캐싱 전략

3. **이미지 저장 공간**: 사용자 증가 시 스토리지 부족
   - **대응**: 클라우드 스토리지 전환 계획

### 비즈니스 리스크
1. **사용자 참여도**: 옷 등록이 번거로워 이탈 가능성
   - **대응**: UX 개선, 자동화 강화

2. **중고거래 신뢰**: 사기 거래 발생 가능성
   - **대응**: 에스크로 시스템(TossPayments), 신고 기능
